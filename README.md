# MIREA-Coursework-Requirements
Курсовая работа: Проектирование системы для рекрутингового агентства
Общая информация
Учебное заведение: РТУ МИРЭА, Институт информационных технологий.

Дисциплина: Обоснование и разработка требований к программным системам.

Тема: Обоснование и разработка требований к ПС поддержки деятельности агентства по подбору персонала.

Студент: Оганнисян Нарек Геворгович (группа ИКБО-63-23).

Оценка за работу: Отлично.

Описание проекта
В рамках этой работы я с нуля спроектировал требования к информационной системе, которая автоматизирует работу кадрового агентства. Сейчас многие небольшие агентства работают в Excel или через почту, что неудобно. Мой проект показывает, как должна выглядеть архитектура системы, чтобы ускорить подбор сотрудников и сделать процесс прозрачным.

Что я сделал в ходе работы:
Анализ «как есть» и «как будет»: Прописал текущие бизнес-процессы агентства и спроектировал новую модель в нотации IDEF0. Это помогло понять, где именно автоматизация сэкономит время.

Проектирование функционала: Определил основные роли в системе (Рекрутер, Клиент-заказчик, Администратор) и что каждый из них может делать.

UML-моделирование: Это основная часть работы. Я нарисовал и описал:

Use Case диаграммы: кто и какие задачи решает в программе.

Sequence и Communication диаграммы: как компоненты системы общаются между собой при выполнении действий.

Class диаграммы: структура кода и логика объектов.

State и Activity диаграммы: жизненный цикл вакансии и кандидата.

Базы данных: Спроектировал схему БД (ER-диаграмма), чтобы данные о кандидатах и вакансиях хранились надежно и к ним можно было писать быстрые SQL-запросы.

Технические требования: Прописал требования к безопасности (защита персональных данных), производительности и удобству интерфейса.

Основные фишки спроектированной системы:
Автоматическая воронка подбора: от первого контакта до оффера.

Хранение истории всех взаимодействий с кандидатами.

Разграничение прав доступа (чтобы рекрутер видел только свои задачи, а админ — всё).

Стек (на который ориентирован проект):
БД: PostgreSQL.

Backend: Java / Python (спроектирована серверная логика).

Архитектура: Трехуровневая (Клиент — Сервер приложений — База данных).
