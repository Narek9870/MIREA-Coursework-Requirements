##  Описание проекта
В рамках работы я с нуля спроектировал требования к информационной системе для автоматизации кадрового агентства. 

**Проблема:** Многие агентства до сих пор используют Excel и почту, что приводит к потере данных и долгому подбору.  
**Решение:** Проектирование четкой архитектуры системы, которая делает процесс найма прозрачным и эффективным.

---

##  Что было сделано в ходе работы

### 1. Анализ процессов
* Прописал бизнес-процессы агентства «как есть» и «как будет».
* Спроектировал модель в нотации **IDEF0**, чтобы найти этапы, которые больше всего нуждаются в автоматизации.

### 2. UML-моделирование (Core)
Это основной технический блок работы. Я подготовил:
* **Use Case диаграммы:** описание ролей (Рекрутер, Клиент, Админ) и их задач.
* **Sequence & Communication:** логика общения компонентов системы.
* **Class диаграммы:** структура кода и объектов.
* **State & Activity:** жизненные циклы вакансии и соискателя.

### 3. Проектирование баз данных
* Разработал **ER-диаграмму** (схему БД).
* Продумал структуру таблиц для хранения данных о кандидатах и вакансиях с возможностью быстрых SQL-запросов.

### 4. Технические требования
* Прописал требования к безопасности (защита персональных данных).
* Определил параметры производительности и удобства интерфейса (UI/UX требования).

---

##  Основные фишки системы
* **Автоматическая воронка подбора:** контроль кандидата от первого касания до оффера.
* **История взаимодействий:** сохранение всех этапов общения с соискателями.
* **Разграничение прав (RBAC):** каждый сотрудник видит только те данные, которые нужны ему для работы.

---

##  Проектируемый стек технологий
* **БД:** PostgreSQL.
* **Backend:** Java / Python (серверная логика).
* **Архитектура:** Трехуровневая (Client — App Server — DB).
